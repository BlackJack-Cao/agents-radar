# Agents Radar 精确搜索与定位设计

## 目标

将现有“仅标记命中日期”的搜索升级为结构化全文搜索。用户输入关键词后，应看到关键词出现在哪一天、哪个报告、哪个章节或条目；点击结果后，页面打开对应报告，滚动到命中内容并高亮关键词。

## 当前问题

当前页面启动时下载全部 Markdown，并把同一天的报告拼接成一段文本。索引只保存 `date -> text`，因此只能判断某个日期是否命中，无法回答：

- 命中哪个报告；
- 命中哪个章节或条目；
- 同一报告中命中了几处；
- 点击后应该滚动到哪里。

同时，历史日报持续增长后，浏览器启动时下载全部 Markdown 的成本也会持续增加。

## 方案

### 1. 构建期生成结构化索引

`pnpm manifest` 在生成 `manifest.json` 和 `feed.xml` 时，同时生成 `search-index.json`。索引按 Markdown 语义块保存记录，而不是保存整篇报告。

每条记录包含：

```ts
interface SearchIndexEntry {
  date: string;
  report: string;
  section: string;
  title: string;
  text: string;
  anchorText: string;
}
```

- `date`：日报日期；
- `report`：报告 ID，例如 `ai-medical`；
- `section`：最近的 Markdown 标题路径；
- `title`：列表项标题、标题行或内容首句；
- `text`：用于匹配和生成摘要的纯文本；
- `anchorText`：从该语义块提取的稳定短文本，用于加载报告后寻找 DOM 元素。

解析时忽略代码围栏、分隔线和空行；标题、列表项、表格行、段落以及 `<summary>` 均作为可搜索语义块。连续的缩进说明归入所属列表项，避免把同一个条目拆成大量零散结果。

### 2. 搜索结果界面

搜索框下方新增结果区域。输入非空关键词后隐藏普通日期导航，显示最多 50 条结果，每条包含：

- 日期；
- 报告名称；
- 所属章节或项目；
- 包含关键词的上下文摘要。

结果按日期倒序排列，同一天内保持报告和原文顺序。关键词使用 `<mark>` 高亮。清空搜索框后恢复原来的日期导航。

为了控制交互噪声：

- 输入使用现有 120ms 防抖；
- 英文搜索不区分大小写；
- 中文按原字符匹配；
- 连续空白统一处理；
- 结果状态显示“匹配 N 条，涉及 M 天”；
- 无结果时显示明确空状态。

### 3. 点击定位

点击搜索结果时：

1. 调用现有 `loadReport(date, report)`；
2. Markdown 渲染完成后，根据 `anchorText` 在标题、段落、列表项、表格行和 `<summary>` 中寻找目标元素；
3. 如果目标位于关闭的 `<details>` 中，先展开对应容器；
4. 在目标元素中高亮当前关键词；
5. 使用平滑滚动将目标放到内容区域上部；
6. 为目标元素添加短暂的定位强调样式。

如果精确锚点因日报格式变化而找不到，则降级为在整篇报告中定位第一个包含关键词的语义块；仍找不到时只打开报告，不阻断阅读。

### 4. 缓存与性能

页面初始化只下载 `manifest.json` 和 `search-index.json`，不再下载全部 Markdown。具体 Markdown 仍在用户打开报告时按需获取，并在浏览器会话内缓存，避免重复请求。

索引保存面向搜索的纯文本，不保存 Markdown 和 HTML。生成过程全部在 GitHub Actions 中完成，不新增服务端、数据库或第三方搜索依赖。

### 5. 发布流程

每日、每周和每月工作流均需：

- 将 `search-index.json` 与 `manifest.json`、`feed.xml` 一起提交；
- 将 `search-index.json` 复制到 GitHub Pages artifact。

旧部署缺少 `search-index.json` 时，页面显示“搜索索引暂不可用”，但报告导航和阅读保持正常。

## 测试

### 单元测试

- Markdown 标题路径和条目解析正确；
- 列表项及其缩进说明合并为一条记录；
- 表格、`<summary>` 和普通段落可以建立索引；
- 代码围栏、分隔线和空内容不会进入索引；
- 中文、英文和特殊字符可以安全序列化；
- 生成的索引按日期与报告顺序稳定输出。

### 页面验证

- 搜索结果显示日期、报告、章节、摘要；
- 点击结果打开正确报告；
- 折叠内容能够自动展开；
- 页面滚动到正确条目并高亮关键词；
- 清空搜索后恢复导航；
- 手机和桌面宽度下结果文本不溢出；
- `search-index.json` 缺失时不影响日报阅读。

## 非目标

- 不增加拼音、同义词或语义搜索；
- 不调用 LLM 处理搜索请求；
- 不引入数据库或搜索服务；
- 不改变日报生成内容；
- 不修改飞书推送逻辑。
